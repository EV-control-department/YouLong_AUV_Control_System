# 开发日志 #1 — 2026-07-05：现状概览

## 项目概况

**YouLong AUV 控制系统** — 面向 SAUVC 竞赛的自主水下机器人控制软件。
基于 ROS 2 Jazzy，混合 ament_python / ament_cmake，双工作空间结构。

---

## 工作空间

```
YouLong_AUV_Control_System/
  workspace_auv/        # AUV 控制栈（无仿真依赖，可直接部署实车）
  workspace_sim/        # 仿真覆盖层（叠加 workspace_auv，加入 Stonefish）
```

仿真模式：`workspace_sim` overlay在 `workspace_auv` 之上，启动时加载 Stonefish 物理引擎 + sim_bridge。

## 各层现状

### Layer 2 — uv_control ║ 算法仿真 ✅ | HIL ✅ | 实机 ❌

**basic_motion** (`/basic_motion` Action Server)：

- 6 种命令：START / SET / WMOVE / BMOVE / WTRAVEL / BTRAVEL
- 坐标系：odom 系（START 时初始化原点），yaw=0° = 初始朝向
- 步进算法：椭圆模型动态步长（STEP_X=0.6m, STEP_Y=0.2m），横向误差指数衰减
- 今日新增：`/basic_motion/pose_info` (PoseInfo, 30Hz)，发布 origin+robot+target 位姿，替代仅仿真可用的 `/auv/state`
- **已验证**：算法仿真中所有 6 种命令走通，HIL 仿真中 MCU 接收 setpoint 并回传状态正常

### Layer 3 — uv_perception　║ 算法仿真 ✅ | HIL ❌ | 实机 ❌

> ⚠️ 今日重写，仅完成算法仿真验证。HIL 和实机完全未测试。

**vision** 节点：
- 仿真模式（`sim_mode:=true`）：订阅 sim_bridge 的拼接双目图 (2560×960)
- 实车模式（`sim_mode:=false`）：cv2.VideoCapture 从 V4L2 设备读取 — **❌ 未验证**
- 拼接图从中切开为左右目各 1280×960，分别跑 YOLO
- 输出 4 路检测 → `/perception/detection/{front,down}_{left,right}`
- 支持去畸变（cv2.undistort，参数可配，默认无效果）— **❌ 未标定**
- 支持数据集采集（`save_dataset:=true`）— **❌ 未验证**
- 支持图像转发调试（`publish_images:=true`）

**position** 节点：
- 订阅 4 路检测 + `/basic_motion/pose_info`
- 4 个摄像头各自独立外参（左右目 y 偏移对称相反）
- 射线历史按 `(class_id, camera_pair)` 分组：front_left+front_right 合流，down_left+down_right 合流
- 多帧单目射线配对交会 → 中值滤波 → 3D 定位
- 输出 `/perception/objects` (10Hz)，带可读类别名
- 队列上限可配置（`max_history`，默认 30）
- **已验证**：仿真环境下检测消息可到达 position，射线队列正常累积，物体位置可成功解算并发布
- **未验证**：定位精度、实车外参是否正确、去畸变后精度

**识别类别**：

### Layer 4 — uv_nav　║ 算法仿真 ❌ | HIL ❌ | 实机 ❌

预留，大概一时半会用不上

NavigatorNode：订阅 `/perception/objects` → 障碍物加入 A* 地图（膨胀半径 2m）→ 航点跟踪 → 调用 basic_motion 执行运动。仅算法仿真验证过，HIL/实机均未测试。

### Layer 5 — uv_task　║ 算法仿真 ✅ | HIL ❌ | 实机 ❌

TaskRunnerNode：JSON 任务列表 → 顺序执行。支持 SET/WMOVE/BMOVE/WTRAVEL/BTRAVEL + start/navigate/wait。通过 ActionClient 调用 basic_motion。仅算法仿真验证过。

## 数据流全景

```
┌─ 仿真 ──────────────────────────────────────────────────────┐
│  Stonefish (4 摄像头) → sim_bridge (hstack 拼接)             │
│    → /auv/front_cam/stitched (2560×960)                     │
│    → /auv/down_cam/stitched (2560×960)                      │
├─ 实车 ──────────────────────────────────────────────────────┤
│  /dev/video0 → cv2.VideoCapture                             │
│  /dev/video2 → cv2.VideoCapture                             │
└─────────────────────────────────────────────────────────────┘
                         ↓
                   vision (YOLO)
        split → undistort → 4 路检测
                         ↓
    /perception/detection/{front,down}_{left,right} (×4)
                         ↓
    /basic_motion/pose_info ──→ position (射线交会)
                         ↓
               /perception/objects (10Hz)
                         ↓
              ┌──────────┴──────────┐
              ↓                     ↓
        NavigatorNode          TaskRunnerNode
        (A* 避障路径)           (竞赛任务决策)
                                    ↓
                              basic_motion Action Server
                                    ↓
                              /zit6/cmd/setpoint
                                    ↓
                              sim_bridge / ZIT6
                              (PID + 推力混合 → 6 推进器)
```
